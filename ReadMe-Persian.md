ابزار **Naabu** یکی از سریع‌ترین و بهینه‌ترین ابزارهای اسکن پورت (Port Scanner) است که توسط تیم ProjectDiscovery به زبان Go توسعه یافته است. این ابزار به دلیل سرعت بالا و مصرف بسیار کم منابع، یکی از محبوب‌ترین گزینه‌ها برای متخصصان تست نفوذ، شکارچیان آسیب‌پذیری (Bug Bounty) و تیم‌های امنیت شبکه است.

در ادامه، تمام پرچم‌ها (Flags) و بخش‌های مختلف راهنمای Naabu را به همراه **مثال‌های کاملاً عملی و قابل تست روی سیستم خودتان** به زبان فارسی آماده کرده‌ام.

> 💡 **نکته مهم قبل از شروع تست:** برای اجرای اسکن‌های نوع SYN (که سریع‌تر و مخفیانه‌تر هستند) نیاز به دسترسی Root یا Administrator دارید. در لینوکس و مک قبل از دستورات خود `sudo` بگذارید و در ویندوز ترمینال (CMD یا PowerShell) را به صورت Run as Administrator باز کنید. همچنین برای تست بدون آسیب رساندن به دیگران، از آی‌پای لوکال (`127.0.0.1`) یا دامنه‌های مجاز مانند `scanme.nmap.org` استفاده کنید.

## ۱. گزینه‌های ورودی (INPUT OPTIONS)

این پرچم‌ها مشخص می‌کنند که چه هدف یا هدف‌هایی باید اسکن شوند.

### پرچم `host-`

* **نمونه دستور:**
```bash
naabu -host 127.0.0.1,scanme.nmap.org

```


* **سناریو و تست عملی:** می‌خواهید سریعاً و بدون ساختن فایل، پورت‌های باز روی سیستم خودتان (`127.0.0.1`) و یک سرور تست قانونی (`scanme.nmap.org`) را بررسی کنید.

### پرچم‌های `list-` یا `l-`

* **نمونه دستور:**
```bash
naabu -list targets.txt

```


* **سناریو و تست عملی:** ابتدا یک فایل به نام `targets.txt` بسازید و چند دامنه یا IP (مثل `localhost` و `scanme.nmap.org`) را خط به خط درون آن بنویسید. سپس این دستور را اجرا کنید تا Naabu تمام آن‌ها را به صورت خودکار و همزمان اسکن کند.

### پرچم‌های `exclude-hosts-` یا `eh-`

* **نمونه دستور:**
```bash
naabu -host 127.0.0.1,scanme.nmap.org -exclude-hosts 127.0.0.1

```


* **سناریو و تست عملی:** شما لیستی از اهداف دارید اما می‌خواهید سیستم لوکال خودتان (`127.0.0.1`) به هیچ وجه اسکن نشود تا ترافیک بیهوده روی کارت شبکه شما ایجاد نگردد.

### پرچم‌های `exclude-file-` یا `ef-`

* **نمونه دستور:**
```bash
naabu -list targets.txt -exclude-file dont_scan.txt

```


* **سناریو و تست عملی:** یک فایل به نام `dont_scan.txt` درست کنید و آی‌پای سیستم خود را درون آن قرار دهید. با این دستور، Naabu تمام اهداف موجود در `targets.txt` را اسکن می‌کند به جز مواردی که در فایل استثنائات قرار دارند.

## ۲. گزینه‌های پورت (PORT OPTIONS)

این بخش مشخص می‌کند Naabu کدام پورت‌ها را باید بررسی کند.

### پرچم‌های `port-` یا `p-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -p 22,80,443,8000-8080

```


* **سناریو و تست عملی:** می‌خواهید روی هدف بررسی کنید که آیا پورت‌های استاندارد وب (۸۰ و ۴۴۳)، پورت مدیریت (SSH ۲۲) و پورت‌های جایگزین توسعه‌دهندگان (۸۰۰۰ تا ۸۰۸۰) باز هستند یا خیر.

### پرچم‌های `top-ports-` یا `tp-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -top-ports 100

```


* **سناریو و تست عملی:** زمان کافی برای اسکن هر ۶۵,۵۳۵ پورت را ندارید. با این دستور، Naabu تنها ۱۰۰ پورت رایج و پرکاربرد اینترنت را بر اساس آمارهای جهانی بررسی می‌کند (گزینه‌های قابل انتخاب: `100`, `1000`, `full`).

### پرچم‌های `exclude-ports-` یا `ep-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -p 1-1024 -exclude-ports 80,443

```


* **سناریو و تست عملی:** می‌خواهید تمام پورت‌های مشهور (۱ تا ۱۰۲۴) را اسکن کنید، اما پورت‌های وب (۸۰ و ۴۴۳) را استثنا کنید چون قبلاً از باز بودن آن‌ها مطمئن شده‌اید.

### پرچم‌های `ports-file-` یا `pf-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -ports-file my_ports.txt

```


* **سناریو و تست عملی:** یک فایل متنی به نام `my_ports.txt` بسازید و پورت‌های خاص خود را (مثلا ۲۲، ۸۰، ۴۴۳، ۳۳۰۶) در آن بنویسید. این دستور برای بررسی دوره‌ای پورت‌های حیاتی سازمان شما بسیار کاربردی است.

### پرچم‌های `exclude-cdn-` یا `ec-`

* **نمونه دستور:**
```bash
naabu -host cloudflare.com -exclude-cdn

```


* **سناریو و تست عملی:** اسکن کردن تمام پورت‌ها روی سرورهایی که پشت Cloudflare یا ابرهای امنیتی هستند بی‌فایده است و باعث مسدود شدن آی‌پی شما می‌شود. این پرچم به صورت خودکار CDN را تشخیص داده و فقط پورت‌های ۸۰ و ۴۴۳ را اسکن می‌کند.

### پرچم‌های `display-cdn-` یا `cdn-`

* **نمونه دستور:**
```bash
naabu -host cloudflare.com -display-cdn

```


* **سناریو و تست عملی:** می‌خواهید خروجی اسکن به شما نشان دهد که آیا پورت‌های باز کشف شده متعلق به یک سرویس CDN (مثل کلودفلر) هستند یا سرور اصلی.

### پرچم‌های `port-threshold-` یا `pts-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -port-threshold 10

```


* **سناریو و تست عملی:** برخی دیوارهای آتش (Firewall) به گونه‌ای تنظیم شده‌اند که به دروغ نشان می‌دهند تمام پورت‌ها باز هستند (Port Spoofing). این دستور به Naabu می‌گوید اگر روی یک هدف بیشتر از ۱۰ پورت باز پیدا شد، اسکن آن هدف را متوقف کن چون فایروال در حال فریب دادن ماست.

---

## ۳. گزینه‌های نرخ ارسال (RATE-LIMIT OPTIONS)

تنظیم سرعت اسکن برای جلوگیری از کرش کردن شبکه یا مسدود شدن توسط فایروال.

### پرچم `c-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -c 10

```


* **سناریو و تست عملی:** اگر سیستم شما قدیمی است یا پهنای باند کمی دارید، تعداد رشته‌های پردازشی همزمان (Worker Threads) را روی ۱۰ تنظیم می‌کنید تا سیستم سنگین نشود.

### پرچم `rate-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -rate 500

```


* **سناریو و تست عملی:** تعداد بسته‌های ارسالی در ثانیه (Packet per Second) را روی ۵۰۰ تنظیم می‌کنید تا اسکن بسیار آرام و چراغ‌خاموش حرکت کند و فایروال‌های حساس را تحریک نکند.

## ۴. گزینه‌های به‌روزرسانی (UPDATE OPTIONS)

### پرچم‌های `up-` یا `update-`

* **نمونه دستور:**
```bash
naabu -update

```


* **سناریو و تست عملی:** قبل از شروع یک پروژه بزرگ تست نفوذ، این دستور را می‌زنید تا ابزار Naabu خود را به آخرین نسخه منتشر شده در گیت‌هاب ارتقا دهد.

### پرچم‌های `duc-` یا `disable-update-check-`

* **نمونه دستور:**
```bash
naabu -host 127.0.0.1 -disable-update-check

```


* **سناریو و تست عملی:** شما در یک محیط ایزوله (بدون اینترنت) در حال تست شبکه داخلی یک بانک هستید. برای اینکه دستور شما به دلیل تلاش برای چک کردن آپدیت اینترنت معطل نشود، این پرچم را فعال می‌کنید.

## ۵. گزینه‌های خروجی (OUTPUT OPTIONS)

ذخیره نتایج برای استفاده در دیگر ابزارها یا ارائه گزارش.

### پرچم‌های `o-` یا `output-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -o result.txt

```


* **سناریو و تست عملی:** اسکن را اجرا کرده و نتایج پورت‌های باز را در فایلی به نام `result.txt` ذخیره می‌کنید تا بعداً آن را بررسی کنید.

### پرچم‌های `j-` یا `json-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -json -o result.json

```


* **سناریو و تست عملی:** می‌خواهید خروجی را وارد یک اسکریپت پایتون یا داشبورد مدیریتی کنید. فرمت JSON Lines ساختار بسیار منظمی برای برنامه‌نویسی به شما می‌دهد.

### پرچم `csv-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -csv -o result.csv

```


* **سناریو و تست عملی:** کارفرمای شما یک فایل اکسل از پورت‌های باز می‌خواهد. با این دستور خروجی را مستقیماً به فرمت CSV تبدیل کرده و در اکسل باز می‌کنید.

## ۶. شناسایی سرویس‌ها (SERVICES-DISCOVERY OPTIONS)

یکی از جذاب‌ترین بخش‌های Naabu که فراتر از اسکن پورت رفته و نوع نرم‌افزار پشت پورت را حدس می‌زند.

### پرچم‌های `sD-` یا `service-discovery-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -p 22,80 -service-discovery

```


* **سناریو و تست عملی:** Naabu علاوه بر گفتن اینکه پورت ۸۰ باز است، در کنار آن کلمه `http` یا برای ۲۲ کلمه `ssh` را چاپ می‌کند تا بدانید چه نوع سرویسی روی پورت در حال دویدن است.

### پرچم‌های `sV-` یا `service-version-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -p 22,80 -service-version

```


* **سناریو و تست عملی:** (نیاز به وجود پروب‌های خدمات) این دستور تلاش می‌کند دقیقاً نسخه نرم‌افزار (مثلاً `Apache 2.4.41` یا `OpenSSH 8.2`) را استخراج کند تا متوجه شوید آیا آن نسخه آسیب‌پذیری عمومی دارد یا خیر.

### پرچم `sV-fast-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -p 80 -service-version -sV-fast

```


* **سناریو و تست عملی:** می‌خواهید نسخه سرویس پورت ۸۰ را بفهمید اما نمی‌خواهید زمان برای تست‌های سنگین و طولانی تلف شود. این پرچم فقط تست‌های بسیار سریع و مرتبط با همان پورت را می‌فرستد.

### پرچم `sV-timeout-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -service-version -sV-timeout 2s

```


* **سناریو و تست عملی:** در یک شبکه داخلی بسیار سریع هستید و اگر پورت تا ۲ ثانیه پاسخ نداد، نمی‌خواهید معطل شوید. زمان تایم‌اوت تشخیص نسخه را از ۵ ثانیه پیش‌فرض به ۲ ثانیه کاهش می‌دهید.

### پرچم `sV-workers-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -service-version -sV-workers 10

```


* **سناریو و تست عملی:** تنظیم می‌کنید که حداکثر ۱۰ پروسه همزمان کار بنر گربینگ (Banner Grabbing) و تشخیص نسخه سرویس‌ها را انجام دهند.

### پرچم `sV-probes-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -service-version -sV-probes ./custom-probes.txt

```


* **سناریو و تست عملی:** شما یک فایل شخصی‌سازی شده از الگوهای شناسایی پورت دارید و می‌خواهید Naabu به جای پایگاه داده پیش‌فرض، از فایل شما استفاده کند.

### پرچم‌های `uP-` یا `udp-probes-`

* **نمونه دستور:**
```bash
naabu -host 127.0.0.1 -p 53,161 -scan-type c -udp-probes

```


* **سناریو و تست عملی:** پورت‌های UDP مانند DNS (۵۳) یا SNMP (۱۶۱) برخلاف TCP با یک دست‌تکانی ساده باز نمی‌شوند. این پرچم پکت‌های اختصاصی پروتکل را می‌فرستد تا سرور مجبور به پاسخ دادن شود.

## ۷. گزینه‌های پیکربندی (CONFIGURATION OPTIONS)

تنظیمات عمیق‌تر شبکه و نحوه رفتار پروتکل‌ها.

### پرچم `config-`

* **نمونه دستور:**
```bash
naabu -config my_config.yaml

```


* **سناریو و تست عملی:** یک فایل کانفیگ اختصاصی با فرمت YAML ساخته‌اید که تنظیمات سرعت و پورت‌ها همیشه ثابت باشد. با این دستور آن را فراخوانی می‌کنید.

### پرچم‌های `scan-all-ips-` یا `sa-`

* **نمونه دستور:**
```bash
naabu -host target.com -scan-all-ips

```


* **سناریو و تست عملی:** دامنه هدف شما پشت یک سیستم موازنه بار (Load Balancer) قرار دارد که به ۴ آی‌پی مختلف اشاره می‌کند. این پرچم تضمین می‌کند که Naabu تک تک آن ۴ آی‌پی پشت دامنه را اسکن خواهد کرد.

### پرچم‌های `ip-version-` یا `iv-`

* **نمونه دستور:**
```bash
naabu -host localhost -ip-version 6

```


* **سناریو و تست عملی:** می‌خواهید سیستم خودتان را صرفاً روی بستر IPv6 اسکن کنید تا مطمئن شوید پورت امنیتی خاصی روی پروتکل جدید اینترنت باز نمانده باشد.

### پرچم‌های `scan-type-` یا `s-`

* **نمونه دستور (نیاز به sudo/Admin):**
```bash
sudo naabu -host scanme.nmap.org -scan-type s

```


* **سناریو و تست عملی:** نوع اسکن را روی `s` (یعنی SYN scan) می‌گذارید که بسیار سریع است و اتصال را کامل نمی‌کند (Half-Open)؛ در نتیجه در بسیاری از لاگ‌های سیستم هدف ثبت نمی‌شود. مقادیر ممکن: `s` یا `c` (مخفف CONNECT).

### پرچم `source-ip-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -source-ip 192.168.1.50

```


* **سناریو و تست عملی:** کارت شبکه شما چند آی‌پی دارد. شما به Naabu دستور می‌دهید که بسته‌ها حتماً از آی‌پی `192.168.1.50` خارج شوند چون این آی‌پی در فایروال سازمان مجاز (White-listed) تعریف شده است.

### پرچم‌های `cp-` یا `connect-payload-`

* **نمونه دستور:**
```bash
naabu -host 127.0.0.1 -p 6379 -connect-payload "PING\r\n"

```


* **سناریو و تست عملی:** می‌خواهید پورت پایگاه داده Redis (۶۳۷۹) را تست کنید. به محض اتصال، کلمه "PING" فرستاده می‌شود تا از رفتار پاسخ سرور متوجه باز بودن واقعی آن شوید.

### پرچم‌های `interface-list-` یا `il-`

* **نمونه دستور:**
```bash
naabu -interface-list

```


* **سناریو و تست عملی:** می‌خواهید نام دقیق کارت‌های شبکه خود (مثل `eth0` یا `wlan0` یا `tun0` مربوط به VPN) را ببینید تا در اسکن‌ها از آن‌ها استفاده کنید.

### پرچم‌های `interface-` یا `i-`

* **نمونه دستور:**
```bash
sudo naabu -host scanme.nmap.org -interface eth0

```


* **سناریو و تست عملی:** شما به یک شبکه VPN متصل هستید اما می‌خواهید این اسکن پورت حتماً از گیت‌وای اصلی کارت شبکه اینترنت شما (`eth0`) خارج شود، نه از داخل تونل VPN.

### پرچم `nmap-cli-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -top-ports 10 -nmap-cli 'nmap -sV'

```


* **سناریو و تست عملی:** یکی از قدرتمندترین ویژگی‌ها؛ Naabu با سرعت بالای خود ۱۰ پورت برتر را اسکن می‌کند و به محض پیدا کردن پورت باز، آن را مستقیماً به ابزار معروف Nmap پاس می‌دهد تا Nmap آپشن `-sV` (تشخیص دقیق سرویس) را فقط روی همان پورت‌های باز اجرا کند. این کار تداخل فوق‌العاده‌ای میان سرعت Naabu و دقت Nmap ایجاد می‌کند.

### پرچم `r-`

* **نمونه دستور:**
```bash
naabu -host target.com -r resolvers.txt

```


* **سناریو و تست عملی:** یک فایل به نام `resolvers.txt` بسازید و آی‌پی دی‌ان‌اس‌های معروف (مثل `8.8.8.8` یا `1.1.1.1`) را در آن قرار دهید. Naabu برای تبدیل نام دامنه‌ها به آی‌پی از این لیست با سرعت بالا استفاده خواهد کرد.

### پرچم `proxy-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -proxy socks5://127.0.0.1:9050 -scan-type c

```


* **سناریو و تست عملی:** می‌خواهید آی‌پای واقعی شما در اسکن مشخص نشود. با این دستور کل ترافیک اسکن شبکه را از پروکسی ساکس ۵ محلی (مثلاً شبکه Tor که روی پورت ۹۰۵۰ باز است) عبور می‌دهید.

### پرچم `proxy-auth-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -proxy socks5://myproxy:1080 -proxy-auth "user:pass" -scan-type c

```


* **سناریو و تست عملی:** استفاده از پروکسی شرکتی که برای عبور از آن نیاز به نام کاربری و رمز عبور دارید.

### پرچم `dns-order-`

* **نمونه دستور:**
```bash
naabu -host target.com -dns-order pl

```


* **سناریو و تست عملی:** ترتیب رزولوشین DNS را مشخص می‌کند. در اینجا ابتدا دی‌ان‌اس‌های غیرفعال/پسیو (`p`) بررسی می‌شوند و در صورت عدم موفقیت به سراغ دی‌ان‌اس‌های محلی (`l`) می‌رود.

### پرچم‌های `sr-` یا `system-resolver-`

* **نمونه دستور:**
```bash
naabu -host target.com -system-resolver

```


* **سناریو و تست عملی:** اجبار ابزار برای استفاده از همان تنظیمات DNS پیش‌فرض سیستم‌عامل شما (فایل `resolv.conf`) جهت حل نام دامنه‌ها.

### پرچم `resume-`

* **نمونه دستور:**
```bash
naabu -resume

```


* **سناریو و تست عملی:** در حال اسکن یک شبکه بزرگ هستید که ناگهان برق یا اینترنت قطع می‌شود. با اجرای مجدد Naabu به همراه این پرچم، اسکن از دقیقاً همان نقطه‌ای که قطع شده بود (با خواندن فایل `resume.cfg`) ادامه پیدا می‌کند.

### پرچم `stream-`

* **نمونه دستور:**
```bash
cat subdomains.txt | naabu -stream

```


* **سناریو و تست عملی:** فعال‌سازی حالت استریم؛ برای زمانی که ورودی‌ها به صورت زنده و بی‌پایان از یک ابزار دیگر تولید می‌شوند و Naabu باید بدون ذخیره در بافر یا مرتب‌سازی، آن‌ها را در لحظه اسکن کند.

### پرچم `passive-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -passive

```


* **سناریو و تست عملی:** **بدون فرستادن حتی یک پکت به سمت هدف!** این پرچم از API رایگان سرویس Shodan InternetDB استفاده کرده و پورت‌هایی که قبلاً در اینترنت از این هدف کشف شده را فوراً به شما نشان می‌دهد. کاملاً قانونی و بی‌صدا.

---

## ۸. کشف هدف (HOST-DISCOVERY OPTIONS)

قبل از اینکه پورت اسکن شروع شود، Naabu بررسی می‌کند که آیا اصلاً سرور هدف روشن است یا خیر.

### پرچم‌های `sn-` یا `host-discovery-`

* **نمونه دستور:**
```bash
naabu -host 192.168.1.0/24 -sn

```


* **سناریو و تست عملی:** می‌خواهید بدانید در شبکه وای‌فای خانه یا شرکت شما چه دستگاه‌هایی روشن هستند. این دستور پورت‌ها را اسکن نمی‌کند، فقط لیست آی‌پی‌های روشن را تحویل می‌دهد.

### پرچم‌های `wn-` یا `with-host-discovery-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -with-host-discovery

```


* **سناریو و تست عملی:** فعال کردن ویژگی کشف هدف زنده قبل از شروع فرآیند اصلی اسکن پورت.

### پرچم‌های `ps-` یا `probe-tcp-syn-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -with-host-discovery -ps 80,443

```


* **سناریو و تست عملی:** برای فهمیدن روشن بودن سرور، یک پکت TCP SYN به پورت‌های ۸۰ و ۴۴۳ می‌فرستد. اگر پاسخ داد یعنی سرور روشن است و حالا اسکن پورت را شروع کن.

### پرچم‌های `pa-` یا `probe-tcp-ack-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -with-host-discovery -pa 80

```


* **سناریو و تست عملی:** ارسال پکت TCP ACK؛ ترفندی برای عبور از فایروال‌های Stateful که پکت‌های SYN را بلاک می‌کنند اما به پکت‌های ACK پاسخ اشتباه (RST) می‌دهند و روشن بودن خود را لو می‌دهند.

### پرچم‌های `pe-` یا `probe-icmp-echo-`

* **نمونه دستور:**
```bash
naabu -host 127.0.0.1 -with-host-discovery -pe

```


* **سناریو و تست عملی:** ارسال پینگ سنتی (ICMP Echo Request) برای بررسی روشن بودن سیستم‌های داخل شبکه.

### پرچم‌های `arp-` یا `arp-ping-`

* **نمونه دستور:**
```bash
sudo naabu -host 192.168.1.1 -with-host-discovery -arp

```


* **سناریو و تست عملی:** در شبکه محلی (LAN) هستید. سریع‌ترین و دقیق‌ترین راه برای مچ‌گیری از سیستم‌های روشن، فرستادن پکت‌های لایه ۲ یعنی ARP است که فایروال ویندوز هم نمی‌تواند جلوی آن را بگیرد.

---

## ۹. بهینه‌سازی (OPTIMIZATION OPTIONS)

### پرچم `verify-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -verify

```


* **سناریو و تست عملی:** برای بالا بردن دقت؛ پس از اینکه اسکن سریع اولیه تمام شد، Naabu پورت‌های بازِ پیدا شده را یک‌بار دیگر با یک اتصال کامل TCP تست می‌کند تا مطمئن شود خطایی (False Positive) رخ نداده است.

### پرچم‌های `ss-` یا `smart-scan-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -smart-scan

```


* **سناریو و تست عملی:** اسکن هوشمند؛ با الگوریتم‌های پیش‌بینی داخلی، اگر چند پورت خاص وب را باز ببیند، حدس می‌زند چه پورت‌های پنهان دیگری ممکن است باز باشند و فقط به سراغ آن‌ها می‌رود تا در زمان صرفه‌جویی شود.

---

## ۱۰. اشکال‌زدایی و گزارش‌ها (DEBUG OPTIONS)

### پرچم‌های `health-check-` یا `hc-`

* **نمونه دستور:**
```bash
naabu -health-check

```


* **سناریو و تست عملی:** اگر Naabu روی سیستم شما ارور می‌دهد یا پکت‌ها ارسال نمی‌شوند، این دستور را بزنید تا وضعیت دسترسی‌ها، کارت شبکه و لایبرری‌های سیستم شما را ارزیابی کند.

### پرچم `debug-`

* **نمونه دستور:**
```bash
naabu -host 127.0.0.1 -debug

```


* **سناریو و تست عملی:** چاپ تمام اتفاقات و لاگ‌های سیستمی زیرپوستی ابزار در ترمینال برای زمانی که می‌خواهید رفتار دقیق برنامه را ردیابی کنید.

### پرچم‌های `verbose-` یا `v-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -v

```


* **سناریو و تست عملی:** به جای اینکه ترمینال تا پایان اسکن ساکت بماند، به صورت زنده پورت‌های باز کشف شده و روند پیشرفت را به شما نشان می‌دهد.

### پرچم `silent-`

* **نمونه دستور:**
```bash
naabu -host scanme.nmap.org -silent

```


* **سناریو و تست عملی:** تمام لوگوها، متن‌ها و آمارهای اضافی حذف می‌شوند و **فقط و فقط آی‌پی و پورت‌های باز** چاپ می‌شوند. عالی برای لوله‌کشی (Pipe) به ابزارهای دیگر:
`naabu -host scanme.nmap.org -silent | httpx`

## ۱۱. گزینه‌های ابری (CLOUD OPTIONS)

این بخش برای اتصال ابزار لوکال شما به پلتفرم ابری ProjectDiscovery Cloud Platform (PDCP) جهت مدیریت متمرکز دارایی‌ها است.

### پرچم `auth-`

* **نمونه دستور:**
```bash
naabu -auth

```


* **سناریو و تست عملی:** این دستور را در ترمینال بزنید؛ از شما کلید API حساب ابری‌تان را می‌خواهد تا سیستم لوکال شما را به پنل تحت وب‌تان متصل کند.

### پرچم‌های `pd-` یا `dashboard-`

* **نمونه دستور:**
```bash
naabu -host target.com -dashboard

```


* **سناریو و تست عملی:** اسکن روی سیستم شما انجام می‌شود اما نتایج آن به صورت زنده آپلود شده و در داشبورد گرافیکی و زیبای وب پلتفرم PDCP به شما و تیم‌تان نمایش داده می‌شود.

### پرچم‌های `aname-` یا `asset-name-`

* **نمونه دستور:**
```bash
naabu -host 10.0.0.5 -dashboard -aname "سرورهای-مالی-مرکزی"

```


* **سناریو و تست عملی:** پروژه اسکن ابری خود را با نام دلخواه برچسب‌گذاری می‌کنید تا در داشبورد وب میان صدها پروژه دیگر گم نشود.
